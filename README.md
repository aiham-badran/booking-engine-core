# 🧠 نواة نظام حجز مواعيد مرنة

**Booking Engine Core**

> نواة مفتوحة المصدر لإدارة منطق حجز المواعيد
> مستقلة عن الأطر – قابلة للتوسعة – مصممة بعقلية هندسية نظيفة

---

## 📌 ما هو هذا المشروع؟

**Booking Engine Core** هو **محرك منطق أعمال (Domain Core)** لنظام حجز مواعيد،
مصمم ليكون الأساس الذي تُبنى عليه:

-   تطبيقات Backend
-   تطبيقات Mobile (Flutter / React Native)
-   منصات SaaS
-   أنظمة مخصصة حسب كل نشاط

❗ هذا المشروع **ليس تطبيقًا جاهزًا**
بل **نواة منطقية نظيفة** تضمن صحة الحجز واستقراره.

---

## 🎯 لماذا هذا المشروع؟

معظم أنظمة الحجز:

-   مرتبطة بإطار عمل
-   صعبة التخصيص
-   منطقها مختلط مع API و UI
-   غير قابلة لإعادة الاستخدام

### هذه النواة تحل المشكلة من الجذور:

✔ منطق حجز واضح وقابل للاختبار
✔ لا تعتمد على Framework
✔ لا تفرض Database
✔ لا تفترض SaaS
✔ مصممة للتوسعة بدون كسر

---

## 🧩 ماذا تقدّم النواة؟

### ✅ الميزات الأساسية (V1)

-   إنشاء حجز
-   إلغاء حجز
-   إعادة جدولة حجز
-   التحقق من التوفر
-   منع تعارض المواعيد
-   سياسات حجز مرنة
-   نظام إضافات (Hooks) قبل/بعد العمليات
-   عزل كامل حسب السياق (Context)
-   اختبارات Domain قوية

---

## 🏗️ المعمارية (Architecture)

النواة مبنية على طبقات واضحة:

```
Client (API / Mobile / Web)
        ↓
     Use Cases
        ↓
      Domain
        ↓
 Repositories & Extensions
```

-   **Domain**: منطق الأعمال
-   **Use Cases**: تنسيق العمليات
-   **Policies**: قرارات قابلة للإعداد
-   **Extensions**: التوسعة بدون تعديل الكود
-   **Repositories**: تجريد التخزين

📖 راجع: `docs/ARCHITECTURE.md`

---

## 🔌 نظام الإضافات (Extensions)

يمكنك التوسعة بدون لمس النواة:

-   إشعارات
-   Logging
-   Analytics
-   Integrations
-   سلوك مخصص لكل Context

كل إضافة:

-   معزولة
-   آمنة
-   لا تكسر النظام

---

## 📱 كيف يستفيد مطورو الموبايل؟

-   النواة تُستخدم في Backend
-   تطبيق Flutter أو غيره يتواصل مع API
-   النواة تضمن:

    -   صحة الحجز
    -   منع التعارض
    -   احترام السياسات

➡️ لا منطق حجز في الموبايل
➡️ لا تكرار قواعد
➡️ مصدر واحد للحقيقة

---

## 🚫 ما الذي لا تفعله النواة؟

هذا المشروع **لن يحتوي على**:

-   REST / GraphQL API
-   Authentication / Authorization Providers
-   قواعد بيانات
-   واجهات مستخدم
-   فوترة SaaS
-   Dashboards

كل ذلك يُبنى **فوق النواة** وليس داخلها.
